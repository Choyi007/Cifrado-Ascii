# Security Policy

## Supported Versions

Currently supported versions:

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅         |
| < 1.0   | ❌         |

## Reporting a Vulnerability

If you discover a security vulnerability in DataVault, please do not publish it publicly.

Instead:

1. Open a private issue or contact the maintainer directly.
2. Include a detailed description of the vulnerability.
3. Include steps to reproduce the issue.
4. Include the version of DataVault being used.

Examples of security issues include:

* Unauthorized access to encrypted data.
* Bypassing authentication mechanisms.
* Data corruption or manipulation.
* Arbitrary code execution.

Reported vulnerabilities will be investigated and fixed as soon as possible.

Thank you for helping improve DataVault security.
